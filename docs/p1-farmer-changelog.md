# FarmFlow P1 升級說明（給農友看的版本）

這次更新做了三件事：**自動接 LINE 訂單**、**自動發送 LINE 通知**、**銀行對帳工具**。下面說明每項怎麼用，以及和你原本流程的差別。

---

## 1. 客戶在 LINE 訂購，會自動進系統

### 之前怎麼做？
你看到客戶在 LINE 傳訂單訊息 → 複製訊息 → 切到 FarmFlow → 貼到「貼上接單」→ AI 解析 → 存檔。

### 現在怎麼做？
**什麼都不用做。** 客戶傳訊息到你的 LINE 官方帳號（OA）後 ~30 秒內：

- **文字訂單**：自動進 `訂單管理` 頁，狀態是「待確認」（草稿）
- **截圖訂單**：客戶連發幾張截圖系統會等 30 秒，全部收齊後合併成一筆草稿
- **客戶識別**：系統會記住這位客戶的 LINE userId。下次他傳訊息系統會直接認得；如果他的 LINE 暱稱跟你本來存的客戶名稱對得起來，系統會把這個 LINE 帳號綁到你既有的客戶資料

### 你要做的事
- 進 `訂單管理` 看新進來的「待確認」訂單
- 點開 → 檢查 AI 解析有沒有錯 → 修正（如果需要）→ 按「已確認」

### 注意
- LINE 訂單會在訂單頁標頭顯示「· 來自 LINE · 客戶 LINE 暱稱」，方便辨識
- 客戶傳貼圖、語音、影片這類訊息**會被自動忽略**，不會建錯訂單

---

## 2. 訂單狀態變更時，會自動推 LINE 給客戶

### 之前怎麼做？
你按下「複製通知文案」→ 切到 LINE → 找到那位客戶 → 貼上 → 送出。

### 現在怎麼做？
**前提**：那位客戶的資料已經綁定 LINE userId（從 P1 起，凡是在 LINE 私訊過你 OA 的客戶都會自動綁好）。

當你在 FarmFlow 把訂單按「已確認 / 已收款 / 已出貨」時，系統會**自動**寄對應的 LINE 訊息給客戶。

### 「複製通知文案」按鈕還在嗎？
**還在**——當作備胎。例如：
- 客戶沒綁定 LINE（你是用其他通路接的單）
- 系統推播失敗想自己重新傳
- 想看一下文案內容會是什麼

它**不會**每次都自動跳——它只是個備用入口。**正常情況下你不用點它**。

### 推播失敗怎麼辦？
推播失敗會跑到 `異常清單` 頁（左側選單），那邊可以按「重試」。

---

## 3. 銀行對帳工具

### 之前怎麼做？
晚上下載銀行 CSV → 開 Excel / 手機看 → 對照 LINE 訊息 → 在訂單上手動勾「已收款」。

### 現在怎麼做？
1. 進 `對帳` 頁，點「上傳 CSV」
2. 選擇你下載的銀行 CSV（目前支援**郵局**、**國泰**兩種格式）
3. 系統自動比對：金額 + 客戶轉帳帳號末 5 碼 → 配對到 FarmFlow 的訂單
4. **綠色**＝自動配對成功，**黃色**＝金額對但帳號末 5 碼不確定，**紅色**＝找不到對應訂單
5. 黃色 / 紅色你手動處理（指定到哪一筆，或標為「忽略」）
6. 確認整批對帳 → 配對成功的訂單自動標 **已收款** → 自動推 LINE 通知客戶

### 配對失敗的怎麼辦？
進 `異常清單` 頁 → 對帳異常 tab。你可以手動指派、或標為「不是這次的款」。

## 4. 異常清單頁面

新增 `/exceptions` 頁（左側選單「異常清單」），統一列出三類需要你注意的東西：

| 類型 | 來源 | 你要做的 |
|---|---|---|
| **低信心訂單** | AI 解析信心 < 70% 的草稿 | 點開逐欄檢查、修正、確認 |
| **推播失敗** | LINE 推送沒成功（access token 失效、客戶封鎖等） | 按「重試」或手動複製文案 |
| **對帳異常** | CSV 對帳沒匹配成功的交易 | 手動指派訂單，或標忽略 |

導覽列右上會顯示總異常數的徽章，**=0 就代表你今天沒事情要處理**。

## 5. 設定確認

P1 啟動需要的設定值（一次性，已協助你設好，但留作備忘）：

- `LINE Channel Secret` 與 `LINE Channel Access Token`（從 LINE Developers Console 取得）→ 填到 `設定` → LINE 整合 tab
- `Webhook URL`：將 `https://farmflow-web.vercel.app/api/line-webhook/<你的 farmer id>` 填到 LINE Developers Console 的 Messaging API 設定
- 銀行帳戶資訊（對帳用）→ 填到 `設定` → 銀行帳戶

如果有看到「LINE 整合：未設定」這類提示，就回那個 tab 確認。

## 6. 已知問題（短期）

詳細列在 [`p1-known-issues.md`](./p1-known-issues.md)。最可能會碰到的：

- **客戶連發多張截圖**：系統等 30 秒才合併處理，所以你看到訂單進系統會比文字訂單慢一點點，這是正常的
- **客戶累計訂單金額**：取消單後不會自動扣減（P2 修），目前用「合併客戶」功能可以全部重算
- **LINE displayName 變更**：如果客戶後來在 LINE 改了暱稱，系統綁定不會自動更新（綁定當下的暱稱會保留）
